在这次作业中，我主要参考了杨老师上传在网络学堂上的2-site的代码和其中使用的库，我的代码在构造初始条件的步骤没有创新，并且使用了杨老师使用的svd函数和缩并函数，这些支持函数在Sub.py中，如果使用这些库不符合作业要求，请告知我，我会立即修改并且使用我本人编写的函数。
可以通过python3 3_site_iTEBD_method.py运行此python脚本，在命令行中会打印符合要求的输出![result](images/result.png "result")，并且生成一个打印了迭代次数和误差的.dat文件以便于调试。

在本程序中使用的three-site链可以写作：
-TC-GC-TA-GA-TB-GB-TC-GC-TA-GA-
  |     |     |     |     |
ABC分别对应数组中的012
three-site和two-site最大的区别就是前者的规范条件更难保持，因此需要小心地做SVD，这也是我在实际编写代码的过程中遇到的最大的问题，起初我没有考虑规范问题，导致程序的收敛性很差。我不仅采用了保持规范的算法，使svd操作与扫描方向相适应，保证还加入了新的sweep以保证在程序进行过程中能够进行更新保证态处于正则规范下，但在实际计算中我发现并不需要单独对态进行规范化，因为态收敛的时候其实已经很接近正则规范了。

本程序的evolution步骤的流程图见
![演化流程图](images/routine.jpg "routine")
核心是合理选择svd的方式使得每次作用演化算符使，除了作用的bond之外其他的bond满足正则规范条件。（这样做可以保证在无限长链中，不满足规范的张量会往一个方向推，不会造成不收敛）
将这次作业的结果和上次作业的结果作对比，可以得到这次作业得到了相同的结果，但是这次作业的收敛速度比较慢，但我认为如果在一开始构造MPS时就构造一个满足正则规范的链可能可以提高收敛效果。除此之外我认为这种方法也许对更加复杂的哈密顿量会有更好的效果，因为并不需要在计算中反复求本征值。
